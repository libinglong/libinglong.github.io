Spring Boot集成Redis

1. 概览

本教程介绍如何在Spring Boot中使用Redis。Redis是一个开源的内存数据结构存储，可用作数据库，缓存和消息代理。本文将提供Spring Boot Redis的核心配置，帮助您更快了解和使用Redis。

2. 依赖配置

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
依赖的版本号应该由Spring Boot Pom提供。

3. Redis核心配置

根据您的Redis实例情况选择以下3种配置方式之一。配置说明参考官方文档

您也可参考源码，在开发时更加有效率，org.springframework.boot.autoconfigure.data.redis.RedisProperties

3.1. Redis standalone模式

一般在开发环境使用，生产环境不使用。 在application.properties中添加如下配置

spring.redis.client-type=lettuce
spring.redis.host=
spring.redis.port=
spring.redis.username=
spring.redis.password=

3.2. Redis sentinel模式

在application.properties中添加如下配置

spring.redis.client-type=lettuce
spring.redis.sentinel.master=
# 格式 host1:port1,host2:port2
# 填写的是sentinel的host和端口，而不是redis的
spring.redis.sentinel.nodes=
spring.redis.sentinel.password=

3.3. Redis cluster模式

在application.properties中添加如下配置

spring.redis.client-type=lettuce
# 格式 host1:port1,host2:port2
# 至少填写一个
spring.redis.cluster.nodes=
spring.redis.username=
spring.redis.password=

相比jedis客户端，lettuce有更好的性能，如果您选择jedis客户端，配置spring.redis.client-type=jedis，此处不再展开介绍
通常您不需要关心以下配置
spring.redis.ssl 默认值false，一般内网不会使用ssl
spring.redis.database 默认值0，尽量不要使用该值做分区处理，因为cluster模式不支持
spring.redis.cluster.max-redirects 默认值5

4. 说在最后

受益于Spring Boot约定大于配置的思想，您只需要填写上述核心配置即可在 生产环境 获得优秀的性能表现。然而您仍然可能希望更精细化的配置，请参考高级配置了解更多。
